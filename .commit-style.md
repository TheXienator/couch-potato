# Commit Message Style Guide

## Format
Keep commit messages **brief and direct** - one short line preferred.

## Pre-Commit Checklist
Before committing TypeScript code:
1. Run `pnpm run build` to check for TypeScript errors
2. Fix any errors before committing
3. Ensure build completes successfully

## Examples

**Good:**
- `Fix render.yaml config`
- `Add health check endpoint`
- `Update dependencies`
- `Remove unused code`

**Avoid:**
- Long explanations in the subject line
- Multiple sentences
- Detailed descriptions (use PR descriptions instead)
